## 摘要
关键帧选择是为长视频问答（QA）提供可验证视觉证据的直接方法。查询的需求各不相同，找到正确的帧取决于知道要查找什么。现有的关键帧选择器要么针对单个查询对每一帧进行评分，要么将查询分解为由单个视觉工具评估的固定模式。我们提出了ToolMerge，一种基于分解与合并的关键帧检索方法：基于大型语言模型（LLM）的规划器将查询分解为工具调用，并指定如何使用布尔运算符合并每个工具的排序。为了直接评估检索性能，我们构建了Molmo-2 Moments (M2M)，这是一个基准数据集，其中的每个问题都通过构造锚定到特定的时间区间。在问答、问题检索和字幕检索任务中，ToolMerge 与先前关键帧选择器相比具有竞争力，尤其是在字幕检索上，比其他方法高出5%。代码和数据可在 https://github.com/michalsr/ToolMerge 找到。

## Abstract
Keyframe selection is a direct way to provide verifiable visual evidence for long-video question answering (QA). Queries differ in what they require, and finding the right frames depends on knowing what to look for. Existing keyframe selectors either score every frame against a single query, or decompose the query into a fixed schema evaluated by a single visual tool. We propose ToolMerge, a keyframe retrieval method based on decomposition and merging: an Large Language Model (LLM) based planner decomposes the query into tool calls and specifies how their per-tool rankings are merged using boolean operators. To evaluate retrieval directly, we construct Molmo-2 Moments (M2M), a benchmark in which every question is anchored to a specific time interval by construction. Across QA, question retrieval, and caption retrieval, ToolMerge is competitive with prior keyframe selectors, most notably on caption retrieval, outperforming other methods by 5%. Code and data can be found at https://github.com/michalsr/ToolMerge .

# 论文总结：Decomposing Queries into Tool Calls for Long-Video Keyframe Retrieval

## 1. 论文核心问题与整体背景
长视频问答（QA）系统需要从大量视频帧中选取关键帧，以便视觉语言模型（VLM）依据帧内容回答问题，并可提供可验证的视觉证据。然而，不同查询对证据的需求各异：有的需要单一场景，有的需要多个物体或实体共存。现有关键帧检索方法存在局限：
- **单查询评分方法**（如SigLIP-Q、BOLT、AKS、WFS）将整个查询文本与每一帧进行相似度匹配，难以处理需要多种视觉线索组合的复杂查询。
- **分解查询方法**（如LIF、T*）将查询分解为固定模式（如物体-关系元组），但因模式僵化而搜索受限。

此外，缺乏能够**直接评估关键帧检索质量**的基准：现有基准问题往往证据帧不唯一，难以精确衡量检索准确性。

**论文目标**：提出一种灵活且高效的关键帧检索方法，并在一个每个问题都有明确时间锚点的基准上进行严格评估。

## 2. 方法论：ToolMerge 分解与合并框架

### 核心思想
将查询分解为多个**独立的工具调用**（使用不同的轻量级视觉工具），然后通过**布尔运算符（AND/OR）** 合并各个工具返回的帧排名，最终选出最相关帧。这个分解与合并过程由一个LLM规划器自动完成，无需训练即可使用，也可通过强化学习（GRPO）进一步优化。

### 流程概览（见图1）
1. **规划阶段**：给定问题与选项（如果有），文本LLM规划器（Qwen3-VL-8B，无帧输入）输出：
   - 多个工具调用的文本查询（可调用SigLIP-2和T-REN）
   - 布尔组合方式（例如`Q1 AND Q2 AND (Q3 OR Q4)`）
2. **执行阶段**：
   - 对视频全帧运行每个工具调用，得到每帧的评分。
   - 对每个工具调用，将评分转换为帧排名（1为最好）。
   - 合并规则：
     - **AND**：取两调用排名的最大值（即较差的排名），确保同时满足所有条件的帧排在前列。
     - **OR**：取最小值（较优排名），扩大匹配范围。
3. **OCR处理**：额外对所有采样帧执行OCR，通过模糊匹配去重和LLM相关性过滤，将包含相关文本的帧以最高优先级插入最终排名中。
4. **帧选择**：对合并排名进行贪心NMS（按排名取帧，且保证帧间时间间隔≥τ秒），选出top-k帧送给下游VLM作答。

### 所用工具
- **SigLIP-2**：用于整体场景的图文匹配。
- **T-REN**：基于区域-文本对齐的物体/实体检索。
- **EasyOCR + GPT-4o-Mini**：用于检测屏幕文本。

规划器的一次输出示例（论文附录F）：
```json
{
  "queries": [
    {"tool": "siglip", "query": "person crossing a river", "id": "Q1"},
    {"tool": "tren", "query": "river", "id": "Q2"},
    {"tool": "tren", "query": "man in yellow jacket", "id": "Q3"}
  ],
  "combine": "Q1 AND Q2 AND Q3"
}
```

### 训练优化
使用**GRPO（Group Relative Policy Optimization）** 微调规划器，以规范化召回奖励（选出的帧落在真实区间内的帧数/贪婪NMS下区间内最大可能帧数）作为奖励，无需运行VLM答案器，从而节省计算并保持检索评估的独立性。

## 3. 实验设计

### 数据集
- **LongVideoBench** 与 **Video-MME**（无字幕）：常规长视频QA基准，评估QA准确率。
- **Molmo2 Moments (M2M)**（自建基准）：
  - 从Molmo-2 Captioning数据集中抽取视频片段描述，每条描述有明确起止时间。
  - 生成多选问题，经8步过滤（包括盲LLM过滤、范围过滤、难度过滤、人工验证等），确保：答案唯一、能从片段内正确回答、难以仅靠均匀采样或纯文本答对。
  - 最终划分：训练集9,677题，验证集748题，测试集756题，视频平均时长19.3分钟，时长范围3分钟~4小时。
- **字幕检索**：直接用源片段描述作为查询，在1,000个描述上评估检索命中率（HIT@K）。

### 评估指标
- QA准确率（%）
- 问题检索HIT@K：前K帧中至少有一帧落在标注时间区间内。
- 字幕检索HIT@K。

### 对比方法
- 盲文本（0帧）
- 均匀采样固定帧数
- **SigLIP-Q**：使用SigLIP-2对整个查询进行帧评分+贪心NMS（强基线）
- 其他关键帧选择器：AKS， BOLT， LIF， WFS， MDP3， AIR（部分仅补充比较）
- Oracle：在真实时间区间内采样（上界）

### 实验设置
- 使用两种VLM答案器：Qwen3-VL-8B和GPT-4o
- 帧预算 K=8 和 K=32
- 所有方法统一使用SigLIP-2-Giant作为相似度模型（BOLT/WFS/AKS），Qwen3-VL用于LIF规划，帧率2fps，温度0。

## 6. 主要发现与结论
- **分解与合并策略有效**：ToolMerge在多个QA和检索任务上超越或持平现有关键帧选择器，尤其在长文本字幕检索上，相对SigLIP-Q提升了约5%（HIT@4）。
- **简单基线很强**：SigLIP-Q（单查询评分+贪心NMS）在问题检索和QA任务中表现与ToolMerge相当甚至略优，成为意想不到的强参考点。
- **工具互补性**：OCR对以字幕为主的Video-MME贡献最大（1.7点），T-REN在视觉色彩丰富的LongVideoBench上贡献显著。
- **M2M基准严格**：盲文本和均匀采样准确率远低于Oracle，验证了M2M能有效衡量检索与QA的关联。
- **GRPO可改善检索**：用召回奖励训练规划器，进一步提升HIT@K和QA准确率。
- **高效**：轻量工具避免了昂贵全帧字幕，检索成本远低于基于密集字幕的方法。

## 7. 论文优点
- **创新性**：首次将LLM规划与多视觉工具、布尔合并相结合用于关键帧检索，灵活且可解释。
- **基准贡献**：构建了严格的时间锚点基准M2M，填补了关键帧检索评估的空白，并公开了完整的构建流程。

- **效率与实用性**：轻量工具+LLM规划实现了低成本高覆盖率检索，并提供了可选的RL微调方案。

## 8. 不足与局限
- **工具集固定**：仅能使用SigLIP-2、T-REN和OCR，无法涵盖音频、精细动作识别等模态，限制了能在文本中表达的查询类型。
- **答案器依赖**：未处理答案推理瓶颈，假设正确帧即可得到正确答案；更高推理难度的题目可能无法获益。
- **M2M范围受限**：问题由标注的描述生成，可能遗漏视频中未描述但客观存在的证据帧；每个问题仅对应单一区间，对于散布全视频的查询不适用。
- **简单基线竞争力强**：SigLIP-Q在多项任务中与ToolMerge几乎持平，表明在部分场景下复杂的分解不一定带来显著增益。
- **计时间隔未深入优化**：τ选择影响性能，但未系统探索与视频类型的关联。
- **GRPO微调数据局限**：训练数据来自M2M自身分布，泛化能力待验证。
