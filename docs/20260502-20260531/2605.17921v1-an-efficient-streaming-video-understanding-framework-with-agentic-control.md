## 摘要
流式视频要求在严格的延迟预算下处理动态的信息密度。然而，现有方法通常采用静态策略，例如固定内存压缩或依赖单一模型，迫使做出权衡：快速模型在复杂查询上失败，而常驻的重型模型违反实时约束且过度复杂化简单查询。我们不是预先固定这些决策，而是提出 R3-Streaming（记忆、响应、推理），将流式视频理解构建为一个级联控制问题：对于每个查询，系统压缩记忆、判断响应就绪度，并顺序路由计算，使得每个下游决策都建立在逐步精化的信息状态之上。为了优化这一流水线，我们引入了一种基于时间感知的遗忘策略用于记忆压缩，因为激进的压缩历史帧可以带来显著的性能增益。对于计算路由，我们提出了 TB-GRPO，一种目标均衡的强化学习目标，将困难查询路由到更强的模型，同时防止模式坍缩。广泛的评估表明，R3-Streaming 在流式多模态大模型中取得了最先进的结果，在 OVO-Bench 上达到 57.92，在 StreamingBench 上达到 76.36，同时将视觉 token 使用量减少了 95% 到 96%。

## Abstract
Streaming video requires handling dynamic information density under strict latency budgets. Yet, existing methods typically employ static strategies, such as fixed memory compression or reliance on a single model, forcing a trade-off: fast models fail on complex queries, while always-on heavy models violate real-time constraints and overcomplicate simple queries. Rather than fixing these decisions upfront, we propose R3-Streaming (Remember, Respond, Reason), which formulates streaming video understanding as a cascaded control problem: for each query, the system compresses memory, judges response readiness, and routes computation sequentially, so that each downstream decision builds on progressively refined information states. To optimize this pipeline, we introduce an age-aware forgetting policy for memory compression, as aggressively compressing historical frames can yield substantial performance gains. For compute routing, we propose TB-GRPO, a target-balanced reinforcement learning objective that routes hard queries to a stronger model while preventing mode collapse. Extensive evaluations demonstrate that R3-Streaming achieves state-of-the-art results among streaming MLLMs, reaching 57.92 on OVO-Bench and 76.36 on StreamingBench, while reducing visual token usage by 95 to 96 percent.

### **论文提出的方法论**

论文提出了一个名为 **R3-Streaming (Remember, Respond, Reason)** 的级联控制框架，该框架由三个顺序耦合的决策模块组成：

*   **核心思想**：将一次流式查询的响应过程分解为三个递进的决策阶段，每个下游决策都建立在上游逐步精炼的信息状态之上，从而实现动态、自适应的资源调度。
*   **关键技术细节**：
    1.  **Remember**：**主动遗忘**策略。
        *   **动机**：基于经验发现，对当前查询最关键的信号高度集中在**邻近帧**，而**历史帧**的 token 虽占用大量注意力，但包含大量冗余甚至噪声。
        *   **实现**：将视频流历史划分为“近期窗口”和“远期历史”两个区域。对近期窗口内的帧采用低压缩率（`τnear≫τhist`）以保留高保真细节，对远期历史采用高压缩率（`τhist→0`）以压缩为紧凑的内存槽，从而消除注意力误导并提供显著的性能增益。此过程无需训练，不引入额外参数。
    2.  **Respond**：**主动响应**机制。
        *   **动机**：在回答前判断已累积的证据是否足够，避免在关键视觉证据出现前过早回答导致幻觉。
        *   **实现**：在快速模型上部署一个轻量级的**就绪头**，该模块将查询和当前记忆状态作为输入，输出概率 `pready`。若概率低于阈值（`pready < 0.5`），则推迟回答并发出 `<Routine>` 信号；否则，进入下一步的推理决策。
    3.  **Reason**：**自适应思考**与 **TB-GRPO** 路由策略。
        *   **动机**：基于模型规模增益在流式任务上并非单调递增的发现，认为“始终调用”重型推理模型是次优的。需要一种机制来动态决定是使用轻量级快速模型直接回答，还是将困难查询升级到重型慢速模型。
        *   **实现**：
            *   **SFT 冷启动**：首先构建训练数据，让快速模型学会输出路由指令（`<Answer>` 或 `<Escalate>`）。对于每个问题，快速模型采样多个回答并由 LLM 评分，平均分低于阈值的问题被标记为需要升级（`<Escalate>`）。
            *   **TB-GRPO**：针对标准 GRPO 算法在二值路由问题上易发生模式坍塌（总是升级）的问题，提出**目标均衡的 GRPO**。通过一个分段惩罚机制，将升级率 `ρ` 锚定在用户指定的目标频段 `[η-γ, η+γ]` 周围。当升级率过高时，抑制升级动作的奖励；当升级率过低时，惩罚直接回答的动作。这确保了在遵守计算预算的前提下，最大化回答质量。
