## 摘要
在线流式视频理解要求模型处理连续视觉输入并实时响应用户查询，其中无界数据流和不可预测的查询时机使得记忆管理成为核心挑战。现有方法通常通过视觉相似性启发式压缩视觉令牌，或在压缩基础上增加KV缓存级别的检索。然而，压缩决策很少融入语义信号，且检索往往在压缩完成后才添加，导致两个阶段难以协调。我们提出SAVEMem，一个无需训练的双阶段框架，将语义感知融入记忆生成，并使检索范围根据查询自适应调整。在第一阶段，SAVEMem在固定记忆预算下在线构建三层流式记忆。固定的伪问题库提供轻量级语义先验，使得长期保留由语义显著性而非仅视觉相似性决定。在第二阶段，SAVEMem在此记忆上执行查询感知检索。锚点条件化的新近门根据查询是针对当下还是遥远的过去，自适应调整检索范围从短期记忆到中长期记忆。在此范围内，查询与记忆令牌之间的延迟交互选择候选帧用于回答。在未训练的情况下应用于Qwen2.5-VL，SAVEMem将OVO-Bench的总分从52.27提升至62.69，并在StreamingBench和ODV-Bench上取得一致增益，同时在128帧时相比主干网络将峰值GPU内存降低48%。

## Abstract
Online streaming video understanding requires models to process continuous visual inputs and respond to user queries in real time, where the unbounded stream and unpredictable query timing turn memory management into a central challenge. Existing methods typically compress visual tokens via visual similarity heuristics, or augment compression with KV-cache-level retrieval. However, compression decisions rarely incorporate semantic signals, and retrieval is often added after compression is finalized, making the two stages hard to coordinate. We present SAVEMem, a training-free dual-stage framework that brings semantic awareness into memory generation and lets the retrieval scope adapt per query. In Stage~1, SAVEMem builds a three-tier streaming memory online under a constant memory budget. A fixed pseudo-question bank provides a lightweight semantic prior, so that long-term retention is shaped by semantic salience rather than visual similarity alone. In Stage~2, SAVEMem performs query-aware retrieval over this memory. An anchor-conditioned recency gate adapts the retrieval scope from short-term to mid- and long-term memory based on whether the query targets the present or the distant past. Within this scope, late interaction between query and memory tokens selects candidate frames for answering. Applied to Qwen2.5-VL without training, SAVEMem improves the OVO-Bench overall score from 52.27 to 62.69 and yields consistent gains on StreamingBench and ODV-Bench, while reducing peak GPU memory by 48\% at 128 frames over the backbone.

- **研究背景**：在线流式视频理解要求模型在视频帧以连续流的形式到来时，能实时响应用户的查询；输入长度无界且查询时机不可预测，这带来了严峻的记忆管理挑战。
- **现有方法局限**：主流方案分为两类——(1) 基于视觉相似度压缩视觉令牌（如 Token Merging、FluxMem）；(2) 在压缩后的 KV 缓存上进行检索（如 ReKV、LiveVLM）。然而，**压缩过程极少利用语义信号**，且**压缩与检索通常被分离设计**，难以协调，导致记忆保留缺乏语义判别力，检索范围也难以按需适配。
- **整体含义**：本文提出 **SAVEMem**，一个**无需训练的双阶段框架**，首次将**语义感知**引入记忆压缩阶段，并让**检索范围随查询自适应**伸缩，从而在有限 GPU 内存下显著提升流式视频问答的性能。

## 2. 方法论

SAVEMem 解耦为 **Stage 1（查询无关的语义记忆生成）** 与 **Stage 2（查询感知的自适应记忆检索）**，两阶段共享同一个基于延迟交互（MaxSim）的评分原语。

### 2.1 Stage 1：三层流式记忆构建
- **语义先验注入**：构建一个固定的**伪问题库**（包含关于物体、计数、动作、场景变化、空间布局等的通用问题），对每一帧的视觉令牌计算相似度 \(s(v) = \max_{q \in Q} \cos(v, q)\)。该分数提供**查询无关的语义显著性**估计，用于指导保留决策。
- **三层记忆结构**（均在恒定令牌预算 \(B\) 下运行）：
  - **短期记忆（Mshort）**：FIFO 缓冲，保留完整令牌，作为最新上下文的锚点。
  - **中期记忆（Mmid）**：当短期窗口溢出时，通过**时间语义剪枝**将最旧帧转移于此，优先保留高 \(s(v)\) 或场景边界的令牌，丢弃与时间近邻高度冗余的低语义令牌。
  - **长期记忆（Mlong）**：当中期溢出时，通过**空间语义选择**将帧压缩至此，以空间覆盖度为硬约束、语义分数为排序依据，保证令牌空间分散且语义显著。
- **全局预算控制**：当三层总令牌数超过预算 \(B\)，持续丢弃长期记忆中语义分数最低的令牌，确保 \(O(1)\) 内存占用。

### 2.2 Stage 2：查询感知的自适应检索
- **锚点条件化新近门**：计算查询 \(q\) 与短期记忆的 MaxSim 亲和度，并与历史亲和度 EMA 比较。若当前亲和度落在既往范围内，则认为查询指向近期，**直接返回短期记忆**，跳过中长期检索；否则触发对岁龄层级的检索。
- **查询感知的延迟交互评分**：当需要检索中长期记忆时，对每个候选帧 \(g\) 计算 ColBERT 风格的帧级相关性：
  \[
  \sigma(g) = \frac{1}{|g|} \sum_{i=1}^{|g|} \max_{q_j \in q} \cos(g_i, q_j)
  \]
- **自适应 Top-K 选择**：根据相关性分数的离散程度，动态决定保留帧数；分数集中时保留更多（证据分散），分数差异大时突出少帧（证据集中）。
- **最终输入**：短期锚点帧无条件保留，与自适应选出的中长期候选帧合并，送入 MLLM 生成答案。

## 7. 优点（亮点）

- **完全训练免费**：即插即用，无需微调，易于集成到现有 MLLM。
- **语义先验轻量且有效**：固定伪问题库仅引入每帧一次 MaxSim 开销，却能显著引导记忆保留。
- **双阶段协同设计**：查询无关压缩与查询感知检索共享评分机制，逻辑清晰，衔接自然。
- **自适应新近门**：基于历史亲和度统计动态决策是否深入检索，避免对短期与长期查询的顾此失彼。
- **内存强可控**：通过三层级联和全局预算实现 \(O(1)\) 内存，帧数增长不影响峰值消耗。
- **消融透彻**：清晰展示各组件独立与联合贡献，结论可信。

## 8. 不足与局限

- **对穷举型查询敏感**：在需要全时间范围覆盖的任务（如计数、复杂因果推理）上仍存在选择性与覆盖性的固有权衡，可能因过滤丢失关键帧。
- **伪问题库固定**：当前库为通用通用探针，未针对特定视频域（如手术、工业检测）进行适配，领域迁移能力未验证。
- **两阶段非联合优化**：训练免费虽方便，但也意味着压缩与检索无法端到端调优，可能留有性能提升空间（文中也指出轻量微调是自然延伸）。
- **实验配置未完全透明**：GPU 型号未公布，推理延迟未测量；虽遵循伪流协议，但真实因果流逐帧推理的延迟影响未评估。
- **长尾语义依赖**：在 Episode Memory 等需要视觉-语义精确对齐的任务上，延迟交互可能对视觉差异大的目标奖励不足，产生微小的性能退化。
