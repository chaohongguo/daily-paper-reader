## 摘要
从驾驶日志中挖掘困难且安全关键的场景受限于缺少难度标签，且没有单一指标（碰撞风险、轨迹模糊性或语义稀缺性）能独立发现此类场景。我们提出 SceneMiner，一个统一的仅依赖摄像头的鸟瞰视图流水线，通过单次前向传播从冻结的视觉语言骨干中输出互补的挖掘信号，无需激光雷达或毫米波雷达：一个用于文本提示场景搜索的检索嵌入、一个多标签场景标签分布和一个连续的基于物理的风险评分（运动预测是副产品，非本文贡献）。构建这样一个多头模型暴露了我们的核心发现——一种我们称为跨任务干扰的失效模式：添加或升级一个任务头会改变共享的激活流并降低参数冻结的兄弟任务头的性能，因此仅冻结参数是不够的。我们的贡献——身份保持多任务微调——通过将每个新子模块初始化为零并冻结馈送到共享流的所有参数来消除这种干扰。从而，挖掘头在仅训练约 102k 参数的同时得以保持比特级一致。场景标签头通过将每个场景聚集为 32 个视觉令牌，在 20 个场景标签上达到 mAP 0.4614（微 F1 0.5557），而嵌入头支持文本提示检索，已定性验证。代码见：https://anonymous.4open.science/r/sceneminer_anonymous-64E5

## Abstract
Mining hard, safety-critical scenes from driving logs is bottlenecked by the absence of difficulty labels, and no single proxy, collision risk, trajectory ambiguity, or semantic rarity suffices to find such scenes on its own. We present SceneMiner, a unified, camera-only bird's-eye-view pipeline that emits complementary mining signals from a frozen vision-language backbone in a single forward pass, with no LiDAR or radar: a retrieval embedding for text-prompted scenario search, a multi-label scene-tag distribution, and a continuous physics-based risk score (a motion forecast is a byproduct, not a contribution). Building such a multi-head model exposes our central finding, a failure mode we term cross-task interference: adding or upgrading one head shifts a shared activation stream and degrades weight-frozen sibling heads, so freezing parameters alone is insufficient. Our contribution, identity-preserving multi-task fine-tuning, removes this interference by zero-initializing every new sub-module and freezing every parameter that feeds the shared stream. The mining heads are thereby preserved bit-identically while training only ~102k parameters. The tagging head reaches mAP 0.4614 (micro-F1 0.5557) on 20 scene tags by pooling each scene into 32 visual tokens, and the embedding head supports text-prompted retrieval, validated qualitatively. Code is available at: https://anonymous.4open.science/r/sceneminer_anonymous-64E5

### 1. 论文的核心问题与整体含义

- **背景与动机**：自动驾驶系统需在长尾分布场景中可靠运行，从海量驾驶日志中挖掘高难度、安全关键场景（如遮挡行人、无保护左转、激进切线等）是实现安全验证与训练数据筛选的关键。然而，现有日志数据缺乏直接的难度标签，且单一的代理指标（碰撞风险、轨迹模糊度、语义稀缺性）均无法独立完成这一挖掘任务。
- **核心挑战**：安全关键场景呈现多面性，因此一个实用的挖掘器需基于同一输入输出多种互补的挖掘信号（检索嵌入、场景标签、风险评分、运动预测），而直接构建多头模型会引发**跨任务干扰**：升级或添加运动预测头会通过共享的激活流（Q-Former 池化后的 tokens）使参数冻结的其他头（标签头、风险头）性能严重退化，打破了“冻结权重即保持功能”的传统认知。
- **整体目标**：本文提出 SceneMiner，一个仅依赖摄像头、无需雷达/激光雷达的统一鸟瞰图（BEV）流水线，旨在用极少的可训练参数（约 102k）同时输出多种互补场景挖掘信号，并通过**身份保持多任务微调**解决跨任务干扰这一根本性失效模式。

### 2. 方法论

#### 2.1 模型架构概览
SceneMiner 包含三大共享阶段和四个并行任务头，仅用单次前向传播即可输出全部信号。

- **共享视觉骨干与 BEV 提取**
  - 使用冻结的 SigLIP2 编码器处理六路环视摄像头图像，得到各摄像头的 patch 特征。
  - 通过 BEVFormer 风格的空间-时间交叉注意力将多摄像头特征提升为 50×50（分辨率 2.048m）的 BEV 网格 `F_bev`。
  - 引入时序堆栈（T=3 帧，时间间隔 0.5s），以 SE(2) 运动补偿拼接后经 1×1 卷积降维。

- **Q-Former 令牌化**
  - 使用 32 个可学习查询通过交叉注意力将三种流（BEV 特征、摄像头特征、运动头产生的 per-agent latent `h`）压缩为 32 个场景 token `Z`，供后续头共享。这一路径正是跨任务干扰的传播媒介。
  - 下游三个头（风险、标签、嵌入）取这些 token 的均值 `z̄` 作为输入。

- **四个并行头**
  - **运动头**（副产品，非贡献重点）：解码器接收 8 个 agent 查询，经位置偏置后输出每个 agent 的潜在向量 `h`，再分解为速度 `va` 和残差轨迹 `ra,t`，组合得到未来 6 步轨迹（Δt=0.5s）。其输出流 `decoded_tokens` 同样馈入 Q-Former。
  - **风险头**：一个 MLP 作用于 `z̄`，经 Sigmoid 输出 [0,1] 连续危险评分，由物理伪标签监督。
  - **标签头**：线性映射至 20 维多标签向量，使用二元交叉熵损失。
  - **嵌入头**：两层 MLP 将 `z̄` 投影为 256 维 L2 归一化嵌入，由同一场景不同关键帧作为正例的 InfoNCE 损失监督。

#### 2.2 核心创新：身份保持多任务微调
针对升级运动头（如增加时序堆栈、位置偏置、解码层和速度子头）时引起的跨任务干扰，提出两个同时满足的条件：

1. **身份初始化**：所有新增子模块（新解码层输出投影、`agent_pos_mlp`、`temporal_proj`、`velocity_head`）均初始化为零，使得扩展后的运动头在起跑时输出完全等于旧头（在残差连接下表现为恒等映射），从而保持 `decoded_tokens` 激活分布不变。

2. **共享流严格冻结**：训练期间仅允许 `velocity_head` 和 `traj_proj`（共计 102,031 参数）更新，其他所有可能影响 `decoded_tokens` 的参数（包括整个运动解码器、Q-Former、三个兄弟头、BEV 编码器等）均被冻结。由于这两个可训练层不在 `decoded_tokens` 的计算图上，保证了共享 token `Z` 在整个微调过程中与旧模型比特级一致（Δ<10^{-4}）。

#### 2.3 伪标签构建（不含人类标注）
- **风险伪标签**：对每个关键帧线性组合三个物理分量——近距指数（0.4 权重）、恒定速度最近碰撞时间（0.3 权重）和弱势道路使用者密度（0.3 权重），再取场景内所有帧的最大值作为场景风险得分。
- **标签富集**：合并 nuPrompt 的目标级提示和官方 Motional `scene.description` 字符串，组成 20 维多标签场景向量。

### 6. 主要结论与发现

- **跨任务干扰**是阻碍统一场景挖掘流水线的关键失效模式：加入或升级一个头会通过共享激活（而非权重变化）令冻结的兄弟头性能骤降。
- **身份保持多任务微调**通过零初始化新子模块并严格冻结馈送共享流的参数，成功消除了这一干扰，使得挖掘头在运动头升级前后达到比特级一致（Δ<1e-4），同时仅需训练～102k 参数。
- SceneMiner 实现的统一流水线能在单次前向传播中输出场景标签（mAP 0.4614）、风险评分（与物理标签 Pearson r = +0.3925）、检索嵌入和运动预测，四种信号互补，为硬场景挖掘提供多维依据。
- 文本提示检索可在不依赖离线跟踪器或 LLM 合成代码的情况下，通过密集潜在嵌入直接匹配，且其排序与 CLIP 互补（0/15 重叠），验证了学习空间的独特性。

### 7. 优点

- **理论洞察新颖**：首次识别并定义了基于激活分布偏移的跨任务干扰，将其与经典的灾难性遗忘、梯度冲突区分开。
- **解决方法精巧且高效**：身份保持微调仅靠设计初始化和冻结规则就达到“零干扰”，无需额外正则项或复杂架构，极为参数高效（可训练参数占全模型十万分之几）。
- **多信号统一于单次前向**：检索、标签、风险、运动轨迹同源输出，摆脱了外部多级流水线，对实时性和部署友好。
- **伪标签可审核且无需人类标注**：物理风险组合与描述子串匹配保证了监督来源的确定性、可审计性，降低了人工标注成本。
- **与现有方案的能力互补**：展示检索结果与 CLIP 不重叠，说明学习到的嵌入空间并非已有视觉语义模型的简单复制，有独立挖掘价值。

### 8. 不足与局限

- **数据集单一**：全部实验仅在 nuScenes 上进行，未在 Argoverse 2、Waymo Open Dataset 等不同分布上验证身份保持微调的泛化性和稳健性。
- **风险监督不独立**：风险头与运动头使用同一套 ground-truth 轨迹注释，预测风险与运动误差的正相关（r=0.233）可能部分源于共享输入，并非完全独立于所设计的物理启发式，缺乏外部风险真值（如人类严重度评分）的验证。
- **检索评估偏定性**：尽管给出 R@10 并说明能力差异，但未建立标准化的 Precision@K、NDCG 评测基准，也未测试开放词汇查询（目前受限于 20 个标签的词汇表），可能会限制在复杂自然语言查询下的实用性。
- **运动头性能并非 SOTA**：作为副产品，其 ADE/FDE 并未与专门的运动预测方法竞争，在更复杂的运动预测场景（如密集交互）中可能效果有限，但作者已明确不作为贡献。
- **应用限制**：伪标签构建依赖于 nuScenes 特有的场景描述和 nuPrompt 提示，迁移至新数据集时需重新设计词汇和物理参数，工作量较大。
