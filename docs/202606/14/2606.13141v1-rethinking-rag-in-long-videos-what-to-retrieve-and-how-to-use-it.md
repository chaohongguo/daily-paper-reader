## 摘要
检索增强生成正从文本扩展到长的自我中心视频，系统必须跨多种模态和时间粒度选择与查询相关的片段。然而，视频RAG的进展受到两个空白的限制：现有基准允许在不使用视频的情况下回答查询，掩盖了检索错误；而先前的方法对每个查询采用单一的模态-粒度配置，忽略了片段级别的变异性。我们通过引入V-RAGBench和CARVE来解决这两个空白。V-RAGBench是一个由⟨查询、证据片段、答案⟩三元组构成的基准，能够对检索和生成进行忠实、解耦的评估。CARVE是一种简单方法，它跨配置运行多个检索器，并采用片段自适应重新排序来为每个片段确定最佳配置。然后，每个片段以其在检索阶段选定的最佳配置进入生成器，产生一种交错的证据形式，其中片段级别的决策贯穿两个阶段。CARVE优于八个最新的视频RAG基线，提供给生成器的片段交错使用多种配置，而非共享单一配置，这是查询级别方法无法实现的行为。

## Abstract
Retrieval-augmented generation is moving beyond text into long, egocentric video, where systems must select query-relevant chunks across multiple modalities and temporal granularities. Yet progress in VideoRAG is limited by two gaps: existing benchmarks allow queries to be answered without the video, obscuring retrieval errors, and prior methods apply a single modality-granularity configuration per query, ignoring chunk-level variability. We address both by introducing V-RAGBench, a benchmark of $\langle$query, evidence chunk, answer$\rangle$ triplets that enables faithful, decoupled evaluation of retrieval and generation, and CARVE, a simple method that runs parallel retrievers across configurations and employs chunk-adaptive reranking to identify the winning configuration for each chunk. Each chunk then enters the generator under its winning configuration selected during retrieval, yielding an interleaved evidence form where the chunk-level decision propagates across both stages. CARVE outperforms eight recent VideoRAG baselines, with the chunks supplied to the generator interleaving multiple configurations rather than sharing a single one, a behavior unattainable by query-level methods.

## 1. 论文的核心问题与整体含义
这篇论文旨在解决长视频（特别是以自我为中心的视频）的**检索增强生成**领域存在的两个根本性挑战：

*   **评估失真问题**：现有的视频问答基准存在严重缺陷，许多查询**无需观看视频即可通过语言先验或常识回答**。这导致即使检索系统完全失败（即提供了无关的视频片段），最终的生成准确率可能依然很高，从而**无法忠实反映检索质量**，使得无法有效诊断和改进系统的检索能力。
*   **方法僵化问题**：当前的VideoRAG方法通常为每个查询固定使用一种**"模态-时间粒度"配置**（例如，统一使用视频级别的视觉特征进行检索和生成）。这忽略了视频片段内容的多样性：对于视觉特征突出的片段，视觉检索更有效；对于语义丰富的片段，文本检索更优。这种"一刀切"的查询级决策无法为不同性质的视频证据块选择最优的表征方式。

因此，本文的核心目标是**重新思考长视频RAG中的两个关键问题："检索什么"和"如何使用"**，通过建立可靠的评估基准和提出更精细化的方法来实现这一目标。

## 2. 论文提出的方法论
论文提出了两个核心贡献：基准数据集 **V-RAGBench** 和方法 **CARVE**。

### V-RAGBench：忠实、解耦的评估基准
该基准由 `<查询, 证据块, 答案>` 三元组构成，其构建过程严格保证了三个关键特性，以使生成准确率与检索质量紧密挂钩：
1.  **证据非重复性**：确保每个查询对应的底层事件在视频中不会以近似重复的方式再次出现，从而每个查询只有一个唯一的正确答案证据块。
2.  **视觉依赖性**：查询的答案无法从查询本身或参数化知识中推断，必须依赖视频视觉内容。
3.  **证据局部化**：答案无法从非证据的视频片段中通过通用视觉线索重构出来，必须定位到特定的证据片段。

### CARVE：块自适应重排序的视频证据检索框架
CARVE 的核心思想是将**模态（视觉 vs. 文本）和时间粒度（帧级 vs. 片段级）的选择权从查询级下放到块级**，并让这一决策同时影响检索和生成阶段。该方法包含两个主要阶段：
*   **第一阶段：并行候选池构建**
    对于给定的视频，每个视频块`v`在离线状态下，预先计算好四种`(模态, 粒度)`配置下的嵌入表征 `ϕ_{m,g}(v)`，形成一个多表征索引。这四种配置构成了方法的搜索空间：
    *   `(文本, 片段级)`: 对整个片段进行文本摘要后嵌入。
    *   `(文本, 帧级)`: 对片段内选取的关键帧分别进行文本描述后嵌入。
    *   `(视觉, 片段级)`: 对整个片段的视觉特征进行嵌入。
    *   `(视觉, 帧级)`: 对片段内关键帧的视觉特征进行池化后嵌入。
    在线处理时，查询`q`会被编码，并在四个独立的表征索引中并行进行检索，每个配置返回其 Top-k 结果，合并、去重后形成一个带有**配置标签（tag）**的候选池`P(q)`。

*   **第二阶段：块自适应重排序**
    此阶段使用一个多模态交叉编码器（Cross-Encoder）`CE`，对候选池`P(q)`中的每个块进行重新打分。关键在于，**每个块`v`仅在其被检索时所用的配置下进行评分**： `s̃(q; (v, (m, g))) = CE(q, ϕ_{m,g}(v))`。
    然后，对于每个唯一的视频块`v`，选择所有标签中得分最高的那个作为其**胜出配置 `(m_v^*, g_v^*)`**。最终，所有块根据它们的胜出得分进行全局排序，形成 Top-k 的最终证据列表。这种机制使得来自不同配置的块可以基于一个可比较的分数进行自由“竞争”和交错排列。
    在生成阶段，每个被选中的视频块都**以它胜出的配置形式**（例如，某些块作为文本，某些块作为关键帧图像）提供给生成器，实现了**模态交织的生成**。

## 3. 实验设计
*   **数据集与基准**：实验主要在作者新构建的 **V-RAGBench** 测试集上进行。该测试集包含来自 **Ego4D 和 EgoLife** 数据集的216个长视频（时长1-9小时不等）的2,100个高质量、解耦的三元组。同时，该基准也按照视频来源（Ego4D/EgoLife）和领域（如居家、社交、户外等）进行了划分，以支持更细粒度的评估。
*   **对比方法**：实验对比了多达八种最新的VideoRAG基线方法，并将其分为两类：
    *   **系统级方法**：对任何查询都采用固定策略，如 **VideoRAG-A** (模态线性插值)、**GQR** (测试时查询优化)、**Freeret** 和 **GME** (单嵌入空间检索)。
    *   **查询级方法**：策略可按查询改变，但对所有检索块仍保持一致，如 **RRF** (查询扩展与排名融合)、**DAT** (动态加权稠密/稀疏检索器)、**VideoRAG-B** (查询分解到多模态数据库)、**UniversalRAG/-LoRA** (训练路由器为查询选择一种最佳配置)。
*   **评估指标**：实现了检索与生成的**解耦评估**。
    *   **检索**：使用 **Recall@K** 和 **nDCG@K** 来衡量证据片段是否被准确且高排名地检索到。
    *   **生成**：使用 **Pass Rate** （通过率），由LLM-as-a-Judge（Qwen模型）判断生成答案与参考答案的关键事实是否一致，考虑了3种不同规模的生成器（Qwen3-VL-8B/32B, Gemma-4-26B）来评估泛化性。

## 4. 资源与算力

*   **消融研究**：系统地研究了“模态-粒度”配置组合的贡献，从单一配置逐步扩展到全部四种，清晰地展示了互补的模态和粒度信息所带来的增益。

## 6. 论文的主要结论与发现
1.  **基准解耦评估的重要性**：V-RAGBench成功实现了检索与生成质量的解耦评估，揭示了高生成准确率可能掩盖差检索问题的现象。
2.  **块级配置选择的必要性**：没有一种单一的“模态-粒度”配置是普遍最优的。最佳配置取决于视频块自身的内容特性，而非查询本身。
3.  **CARVE方法的优越性**：通过块自适应重排序，CARVE在检索和生成任务上均显著优于所有系统级和查询级的基线方法。其性能提升不仅源于更好的检索，更在于将检索阶段选择的胜出配置**延伸到生成阶段**，形成模态交织的证据输入。
4.  **高效的决策机制**：CARVE通过只为每个最终选定的块渲染其胜出配置，在取得最佳性能的同时，其在线延迟（4.6秒）低于单纯使用视觉配置的方法，实现了性能与效率的良好平衡。

*   **方法简洁有效**：CARVE无需训练，思路清晰，它将文本RAG中成熟的重排序技术巧妙地改造为视频领域的**块级配置决策引擎**，具有很强的解释性和实用性。
*   **实验评估全面且有力**：从多维度（检索/生成）、多方法（8+基线）、多模型（3种生成器）、多角度（消融、内部对比、分布分析）进行的实验，提供了丰富且极具说服力的证据。

## 8. 不足与局限
论文作者在附录中A章节也坦诚地讨论了其局限性：
1.  **配置空间有限**：当前仅探索了两种模态（视觉、文本）和两种粒度（帧级、片段级）的组合，共四种配置。虽然足以证明方法的有效性，但尚未探索更丰富的模态（如音频）和更细的粒度。
2.  **局限于一次性检索**：CARVE保持了标准的一次性“检索-生成”流程，没有探索迭代式或智能体式的（Agentic）多轮检索，后者可能在复杂查询上有更大潜力。
3.  **领域偏向性**：V-RAGBench专为自我为中心的长视频（Ego4D/EgoLife）设计，这虽然是一个极具挑战和现实意义的目标，但其结论是否直接适用于叙事性更强的电影、讲座或指令性视频等结构化视频，还有待考证。
4.  **隐私与社会影响**：自我中心视频天然涉及个人隐私，虽然论文本身仅创建了研究基准，但其潜在应用（如个人记忆增强）必须考虑隐私保护和数据安全等伦理问题。
