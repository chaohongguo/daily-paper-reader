## 摘要
CoVR-R研究推理感知的组合视频检索：给定一段参考视频和一条编辑指令，系统必须检索出满足编辑要求的目标视频。主要难点在于目标并非直接描述，必须从物体身份、动作顺序、最终状态、手部交互以及场景转换等细粒度变化中推断出来。我们基于Qwen3.5-27B构建了一个零样本的先推理后检索流程。对于图库中的每个视频，模型生成一段面向检索的结构化描述，并通过带token依赖权重的生成token隐状态池化得到稠密嵌入向量。对于每个查询，模型首先对参考视频和指令执行编辑推理，然后生成目标视频描述，其隐状态用作查询嵌入。我们用一个作用于生成文本的TF-IDF分支补充稠密检索，并用分割特定的权重融合两种排序结果。在验证集上，当前最佳提交达到了R@1 80.81、R@5 94.86、R@10 97.11、R@50 98.59的成绩。在盲测分割上，R@1为89.73，R@5为95.79，R@10为96.63，R@50为97.98。

## Abstract
CoVR-R studies reason-aware composed video retrieval: given a reference video and an edit instruction, the system must retrieve the target video that satisfies the edit. The main difficulty is that the target is not described directly; it must be inferred from fine-grained changes in object identity, action order, final state, hand interaction, and scene transition. We build a zero-shot reason-then-retrieve pipeline around Qwen3.5-27B. For each gallery video, the model generates a retrieval-oriented structured description and a dense embedding by pooling generated-token hidden states with token-dependent weights. For each query, the model first performs edit reasoning over the reference video and instruction, then generates a target-video description whose hidden states serve as the query embedding. We complement dense retrieval with a TF-IDF branch over the generated texts and fuse the two rankings with split-specific weights. On validation, the current best submission reaches 80.81 at R@1, 94.86 at R@5, 97.11 at R@10, and 98.59 at R@50. On the blind test split, it reaches 89.73 at R@1, 95.79 at R@5, 96.63 at R@10, and 97.98 at R@50.

### 1. 研究动机与核心问题

- 任务定义：CoVR-R（Composed Video Retrieval with Reasoning）要求系统根据一段参考视频和一条文本编辑指令，从一个大型图库中检索出符合编辑要求的目标视频。
- 核心挑战：目标视频从未被直接描述，必须从细粒度的变化中推断，例如物体身份替换、动作顺序调整、最终状态差异、手部交互变化以及场景转换。
- 实际困难：
  - 通用字幕式表征往往将多阶段动作压缩为简短场景摘要，丢失检索所需的关键证据。
  - 纯词汇匹配在面对需要推断的目标时脆弱（目标通常未被显式命名）。
- 整体思路：采用“先推理后检索”（reason‑then‑retrieve）策略，先理解编辑应带来的变化，再生成与检索目标对齐的表征，最后融合稠密与稀疏检索。

### 2. 方法论

#### 2.1 核心理念
- 表征必须保留动作顺序、最终状态和转换线索，而非单纯追求字幕质量。
- 在库端和查询端都生成面向检索的结构化描述，并从中提取稠密嵌入；再辅以 TF‑IDF 稀疏检索，通过分数融合互补。

#### 2.2 库端流程
- 使用 Qwen3.5‑27B 为每个库视频生成紧凑的、检索导向的描述，提示词强制保留：主体对象、相关属性、动作链、手部交互、最终状态、场景/镜头切换以及干扰细节。
- 从生成的描述中计算稠密向量：
  - 对每个生成的 token i，取其隐状态 hi，并赋予一个重要性权重 wi（内容词如对象、动作、状态、转换等权重较高，停顿词/标签权重低）。
  - 池化公式：g = (∑ wi hi) / (∑ wi)，然后进行 L2 归一化。

#### 2.3 查询端流程
- 输入：参考视频 + 编辑指令。
- 第一步：编辑推理——模型分析哪些内容应保持不变、哪些应消失、新目标动作或状态应如何出现。这一中间推理步骤减少了直接将源内容复制到目标描述的风险。
- 第二步：生成目标视频描述，并应用与库端相同的权重池化策略得到查询嵌入 q。
- 相较于直接嵌入原始指令，此方法与真实检索目标的距离更近，能更好地捕捉多步编辑。

#### 2.4 稠密检索
- 使用余弦相似度排序：s_dense(q, g) = (q⊤g) / (∥q∥ ∥g∥)。

#### 2.5 稀疏检索与融合
- 额外构建 TF‑IDF 分支：对生成的查询描述和库描述计算 TF‑IDF 得分（使用 scikit‑learn 实现）。该分支对精确名词、材质、状态变化等有良好区分度。
- 最终得分：s_final = α·ŝ_dense + (1‑α)·ŝ_tfidf，其中 ŝ 为归一化分数。
- 融合权重 α 根据数据集分割（WebVid 与 Something‑Something V2）单独设置，以适配不同的错误模式。
