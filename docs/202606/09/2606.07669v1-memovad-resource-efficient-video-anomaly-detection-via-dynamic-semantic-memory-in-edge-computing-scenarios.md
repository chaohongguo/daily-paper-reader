## 摘要
在现实世界监控中部署视频异常检测（VAD）面临一个根本性矛盾：既需要高层次语义来保证有效性，又受限于边缘设备有限的计算资源。视觉语言模型（VLM）提供了丰富的开放词汇语义，但其延迟和计算成本使得无法在设备端部署。为解决这一挑战，我们提出MemoVAD，一个边缘-云协同框架，选择性地将VLM语义融入流式VAD。MemoVAD在边缘运行大部分推理，采用轻量级检测器和因果时序上下文编码器（TCE）来建模时间依赖。具体来说，我们引入一种基于主观逻辑的不确定性感知门控（UAG）策略，对感知不确定性进行建模，并仅针对高不确定性和语义新颖的视频片段查询云端VLM。此外，我们设计了一个动态语义记忆（DSM），缓存VLM验证的原型以实现高效检索，使边缘模型通过语义适配器逐步融合VLM级别的语义。在UCF-Crime和XD-Violence数据集上通过真实边缘设备进行的实验表明，MemoVAD大幅减少了通信开销，同时超越了最先进的性能。

## Abstract
Deploying Video Anomaly Detection (VAD) in real-world surveillance faces a fundamental tension between the demand for high-level semantics to ensure effectiveness and the limited computational resources of edge devices. Vision-Language Models (VLMs) provide rich open-vocabulary semantics, but their latency and computational cost preclude on-device deployment. To address the challenge, we propose MemoVAD, an edge-cloud collaborative framework that selectively incorporates VLM semantics into streaming VAD. MemoVAD runs most inference on the edge with a lightweight detector and a causal Temporal Context Encoder (TCE) to model temporal dependencies. Specifically, we introduce an Uncertainty-Aware Gating (UAG) policy grounded in Subjective Logic to model perceived uncertainty and query the cloud-based VLM only for high-uncertainty and semantically novel clips. Besides, a Dynamic Semantic Memory (DSM) is designed to cache VLM-verified prototypes for efficient retrieval, enabling the edge model to progressively incorporate VLM-level semantics via a semantic adapter. Experiments on UCF-Crime and XD-Violence datasets via a real edge device show that MemoVAD substantially reduces communication overhead while surpassing state-of-the-art performance.

### 2. 方法论：核心思想与关键技术细节
MemoVAD由三个核心组件构成，形成“边缘推理为主、云端VLM按需查询、知识缓存复用”的协同范式。

#### 2.1 边缘驻留检测器（轻量化）
- **视觉骨干**：采用冻结的**VideoMAE‑Small**提取视频片段的运动特征向量 \( x_t \)。
- **时序建模**：引入轻量级**因果时序上下文编码器（TCE）**，为一个2层因果Transformer，从固定长度的历史窗口中聚合时序信息，输出预增强的特征 \( h_t^{\text{pre}} \)。推理时使用KV缓存实现流式高效更新。

#### 2.2 不确定性感知门控（UAG）
- **目标**：仅在边缘模型**证据不足**或**样本语义新颖**时触发云端VLM查询，避免无谓通信。
- **实现**：
  - 使用**主观逻辑（Subjective Logic）** 建模证据：门控头输出非负证据向量 \( e_t \)，形成狄利克雷分布 \( \text{Dir}(\alpha_t) \)，其中 \( \alpha_t = e_t + 1 \)。
  - 感知不确定性定义为 \( u_t = 2 / (\alpha_{t,0} + \alpha_{t,1}) \)，即证据总量的倒数。
  - 同时计算当前特征与动态语义内存中键的最大余弦相似度 \( s_{\text{retrieval}} \)。
  - **触发条件**：\( \text{Trigger}_t = \mathbb{I}( u_t > \tau_{\text{unc}} \land s_{\text{retrieval}} < \tau_{\text{sim}} ) \)，即**高不确定且低记忆匹配**时才请求VLM。

#### 2.3 动态语义内存（DSM）
- **结构**：键值对内存 \( \mathcal{M} = \{ (k_i, v_i) \} \)，键为边缘特征 \( h^{\text{pre}} \)，值为VLM的语义嵌入 \( z^{\text{vlm}} \)。
- **检索与融合**：
  - 通过**多头交叉注意力**从记忆中检索语义信息 \( m_t \)。
  - 设计**语义适配器模块（SAM）**：将 \( m_t \) 投影后与边缘特征拼接，经瓶颈MLP处理，再通过带可学习门控α的残差连接融合，得到最终特征 \( h_t^{\text{enriched}} \)。
- **在线更新**：当VLM确认异常时，将新原型加入记忆；当记忆满时，通过**在线语义多样性替换**策略移除最冗余的键（与所有其他键的最大余弦相似度最大者），以保持原型的多样性。

#### 2.4 训练目标（弱监督视频级标签）
- **MIL排序损失**：基于多实例学习，迫使异常视频的最大片段得分高于正常视频，并留出间隔 \( m \)。
- **语义蒸馏损失**：仅在VLM被触发时，最小化边缘富裕特征经投影头与冻结VLM语义嵌入的L2距离。
- **时序平滑损失**：鼓励相邻片段预测得分的连续性，并施加稀疏惩罚以避免全高分平凡解。
- **门控监督损失**：采用证据深度学习损失，仅对每包中得分最高的片段应用。
