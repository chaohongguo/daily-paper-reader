## 摘要
基于环境传感器的人体活动识别（HAR）使智能家居应用成为可能，如健康监测和辅助生活。然而，在实际部署中，传感器事件作为连续流到达，活动边界未知。因此，滑动窗口推断会产生许多跨越过渡且包含混合活动的窗口，导致边界污染，违反了大多数基准和模型所使用的预分割实例假设。此外，许多管道未充分利用空间上下文，将传感器 ID 视为独立标记。我们提出 LastAct，一个面向流式智能家居 HAR 的轨迹中心框架，它在处理混合窗口时针对最近的活动，同时显式建模空间结构。LastAct 将传感器事件投影到家居平面图上，形成保留空间连续性的布局对齐轨迹图像序列。一个轻量级门控识别受污染的窗口，边界定位器估计最后的过渡点，以实现边界引导的掩码，强调过渡后证据并抑制陈旧上下文。为了提高效率，我们复用预计算的布局对齐模板缓存，避免重复渲染。在四个公共智能家居数据集上，采用近乎现实的混合活动协议进行实验，LastAct 在纯净窗口上达到了具有竞争力或更优的性能，并在交叉/混合窗口上大幅提升了宏观 F1 分数，展示了在近乎现实的滑动窗口机制下的鲁棒性提升。

## Abstract
Human Activity Recognition (HAR) from ambient sensors enables smart-home applications such as health monitoring and assisted living. In realistic deployments, however, sensor events arrive as a continuous stream and activity boundaries are unknown. Sliding-window inference therefore produces many windows that straddle transitions and contain mixed activities, creating boundary contamination that violates the pre-segmented instance assumption used by most benchmarks and models. Moreover, many pipelines under-use spatial context by treating sensor IDs as independent tokens. We present LastAct, a trajectory-centric framework for streaming smart-home HAR that targets the most recent activity under mixed windows while explicitly modeling spatial structure. LastAct projects sensor events onto the home floorplan to form a layout-aligned trajectory image sequence that preserves spatial continuity. A lightweight gate identifies contaminated windows, and a boundary localizer estimates the last transition to enable boundary-guided masking that emphasizes post-boundary evidence and suppresses stale context. For efficiency, we reuse a precomputed layout-aligned template cache to avoid repeated rendering. Empirically, across four public smart-home datasets under near-realistic mixed-activity protocols, LastAct achieves competitive or superior performance on pure windows and yields substantial Macro-F1 gains on cross/mixed windows, demonstrating improved robustness under near-realistic sliding-window regimes.

## 1. 核心问题与整体含义

- **研究背景**：基于环境传感器的人体活动识别（HAR）在智能家居健康监测、老年护理、辅助生活等领域具有重要应用价值。传统方法大多依赖预分割的活动实例（即有明确起止标注的干净窗口），假设每个滑动窗口仅包含单一活动。
- **核心问题**：实际部署中，传感器事件以连续流形式到达，活动边界未知。滑动窗口推理频繁捕获活动过渡，产生包含多个活动片段的**边界污染**（交叉活动窗口）。这违反了传统模型的预分割假设，导致分类性能剧烈下降。此外，大量方法将传感器ID视为独立标记，严重**忽视空间上下文**，未充分建模住户在物理空间中的移动轨迹。
- **整体含义**：提出**LastAct**，一个以轨迹为中心的流式HAR框架，旨在从连续传感器流中**实时定位并识别最近的活动**。它显式建模空间结构，并通过边界感知推理处理混合窗口中的污染问题，从而提升在近乎现实部署条件下的鲁棒性。

## 2. 方法论

### 2.1 问题形式化
- 定义事件流 $E$，窗口长度 $w$，最新活动识别目标标签为窗口最后一个事件对应的活动 $y^{(k)}$。
- 引入**活动纯度** $\rho^{(k)}$ 量化边界污染程度（100% 纯度为单一活动，低纯度则尾部仅含少量最新活动事件）。训练目标是在 $\rho^{(k)}$ 下降时保持识别鲁棒性。
- 构建**Episode-Aligned Pure Windows**（仅含单一活动）和**Near-Realistic Cross-Activity Windows**（以受控纯度 $p$ 混合目标活动与前序上下文），为系统化评估提供基础。

### 2.2 轨迹表示与活动主干网络
- **布局对齐轨迹图像序列**：
  - 将传感器映射到家庭平面图的像素坐标，缩放到 $R \times R$ 画布。
  - 为每个传感器预渲染颜色分明的模板（缓存），构建可重用模板库 $B$。
  - 对每个事件窗口，按时间步组合模板，形成 $w \times 3 \times R \times R$ 的轨迹图像序列，保留空间布局与移动顺序。
- **循环时间编码**：将时间戳分解为小时、星期等周期性分量，用正弦/余弦编码映射为连续特征，消除时段边界突变。
- **活动主干网络**：
  - 用 CNN 处理每帧轨迹图像，串联时间编码，经单向 LSTM 建模序列依赖，注意力池化生成窗口级表示，最后通过全连接层分类。
  - 仅在 Pure Windows 上训练，学习干净的轨迹模式。

### 2.3 轨迹引导的近乎现实活动识别框架
- **交叉窗口门控机制**：从 CNN 提取的空间特征序列计算均值、标准差和 Top-K 帧间差异的统计量，输入 MLP 二分类器判断窗口为纯窗口还是交叉窗口。
- **特征增强的边界检测器**：
  - 对交叉窗口，冻结的主干特征经 Transformer Encoder 建模全局依赖，输出每个时间步是边界的概率。
  - 设计复合损失函数：
    $$
    \mathcal{L}_{\text{bound}} = \mathcal{L}_{\text{BCE}} + \lambda_{\text{card}} \mathcal{L}_{\text{card}} + \lambda_{\text{last}} \mathcal{L}_{\text{KL}} + \lambda_{\text{peak}} \mathcal{L}_{\text{peak}}
    $$
    其中 $\mathcal{L}_{\text{BCE}}$ 为加权二分类交叉熵；$\mathcal{L}_{\text{card}}$ 约束边界数量；$\mathcal{L}_{\text{KL}}$ 以高斯分布引导最后边界位置；$\mathcal{L}_{\text{peak}}$ 鼓励概率分布尖锐化。
- **边界感知活动预测**：
  - 纯窗口：直接送入分类头。
  - 交叉窗口：利用估计的最后边界 $\hat{\tau}_{last}$ 构建软注意力掩码，掩码中边界前的权重接近零，边界后为 1。用引导掩码聚合特征，使分类器仅基于最新活动片段进行预测。

### 2.4 效率优化
- **轨迹记忆库**：预计算布局缓存，推理时只需模板查表与合成，避免逐事件重复渲染。

## 3. 实验设计

### 3.1 数据集与场景

### 3.2 评价协议
- **分层交叉验证（3-fold）**：按活动标签分层划分，纯净窗口训练，混合窗口测试。
- **时间顺序划分（Day-Based）**：按日历日期排序，前60%天为训练，中20%验证，后20%测试，避免未来信息泄露，更接近真实部署。
- **原始流滑动窗口协议**：直接在连续事件流上以固定步长（主要为步长1）生成测试窗口，**不使用**活动边界标注进行窗口构建，仅用于评估标签分配，真实模拟在线推理。

### 3.3 对比方法

所有方法均保证**公平比较**：主干网络仅用纯窗口训练，测试时用相同的混合窗口或原始流窗口。

## 6. 主要结论与发现
- **混合窗口性能大幅领先**：在 Milan、Aruba、Kyoto 上，LastAct 的 Macro-F1 相比最强基线提升 15-25 个百分点；在 Orange 上长窗口时超越基于文本的 TDOST。受益于边界感知推理和空间轨迹。
- **原始流滑动窗口（无边界标注）仍保持最优**：所有数据集上聚合 Macro-F1 均为最高，证明其在部署式推理中的有效性。
- **低纯度窗口优势显著**：在 10-30% 纯度区间（最新活动仅占窗口很小部分），Macro-F1 达 47%-55%，而基线普遍低于 20%，说明边界引导掩码有效抑制陈旧上下文。
- **纯窗口主干网络本身具有竞争力**：在传统预分割协议下与最强基线持平或略优，说明框架的增益并非来自单纯替换更强的主干，而是来自门控和边界感知推理。
- **边界检测准确且下游有效**：在多数数据集上最后边界定位准确（±2 误差内 70-85%），使用检测边界替代真值边界时 Macro-F1 下降有限（尤其在活动较长时），表明检测器可靠。
- **时空泛化表现突出**：在时间顺序划分下，低纯度和中纯度窗口的 Macro-F1 提升高达 20-60 个百分点；对适度空间噪声鲁棒，但大幅扰动会造成性能下降，证明了空间信息确实被利用。
- **轨迹表示与时序编码贡献**：颜色和形状影响微小，但颜色编码有助于区分传感器类型；循环时间编码收益因数据集而异，默认小时+工作日提供稳定小幅度增益。

## 7. 优点（方法与实验设计的亮点）
- **首次系统化定义和评估边界污染**：提出活动纯度概念，设计了受控的交叉窗口构建和评估协议，为领域提供了可复现的测试基准。
- **创新的轨迹中心表示**：将离散传感器序列转化为布局对齐的图像序列，保留了空间连续性和移动路径，优于传统的 ID 嵌入或人工二维矩阵。
- **模块化三阶段推理框架**：纯/交叉门控 → 最后边界检测 → 掩码聚合分类，解耦了活动识别与边界定位，便于按需调用，计算高效。
- **复合损失精细指导边界定位**：通过基数约束、KL 散度聚焦最后边界、峰值正则化，显著提升了边界检测精度和下游分类表现。
- **工程友好的效率设计**：模板缓存机制使得轨迹构建仅需查表与合成，减少了重复计算，适合流式环境。
- **实验周密全面**：多数据集、多窗口大小、分层与时间划分、详尽消融、定性分析，所有对比均在同等条件下进行，结论可信度高。
- **对实际部署挑战的深入探讨**：包括原始流滑动窗口、未来日泛化、空间扰动，体现了对真实场景的考量。

## 8. 不足与局限
- **算力信息缺失**：未报告 GPU 型号、数量及训练耗时，影响可复现性评估。
- **空间元数据依赖**：需要传感器到房间或平面图的空间映射，在完全没有布局信息或传感器频繁移动的环境中部署受限。虽然可以近似构建伪坐标，但可能降低效果。
- **跨家庭泛化未探索**：评估仅在各家庭内部进行，模型可能无法直接迁移到不同布局、不同传感器类型、不同住户习惯的新环境中。
- **监督训练需求**：假设有标注的目标家庭数据用于训练，未来需减少对标注数据的依赖（如少样本、域适应）。
- **仅支持单标签最新活动识别**：不能处理同时发生的多活动或并行活动场景。仅在 Kyoto7 上做了初步的并行检测可行性实验，并未解决多标签、分人的识别问题。
- **并行活动实验局限性**：仅做了二元分类（单活动 vs 并行），未扩展到多标签活动识别或分人归属。
- **极端类别长尾问题**：对于极少样本活动（如 `Meditate`、`Eve_Meds`），所有方法包括 LastAct 仍表现不佳，边界感知无法完全弥补数据稀疏。
- **评估仍为受控近似**：纯度桶和固定窗口长度为系统压力测试而设，实际流中的窗口分布可能更为复杂，存在微小评估偏差。
